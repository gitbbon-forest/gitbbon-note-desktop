# [Design] Gitbbon 앱 및 확장 프로그램 아이콘 디자인 요청

## 📋 개요 (Overview)
Gitbbon 프로젝트의 **Gitbbon Manager (VS Code Extension)** 및 **Gitbbon Note (Application)** 에서 사용할 아이콘 디자인을 요청합니다.

## 🎨 디자인 컨셉 및 방향
- **키워드**: Git, Branch, History, Note, Productivity
- **스타일**: 심플함, 가독성 높음, 모던함
- **테마 호환성**: VS Code의 다크 테마와 라이트 테마 모두에서 잘 보여야 합니다.

## 📏 규격 및 제출 포맷 (Specifications)

### 1. 마스터 파일 (Master File)
모든 파생 이미지의 원본이 되는 고해상도 벡터 파일입니다.
- **포맷**: AI, EPS, 또는 SVG
- **크기**: 1024x1024 px 이상

### 2. VS Code 확장 프로그램용 (Extension Icon)
마켓플레이스와 확장 프로그램 탭에서 사용됩니다.
- **포맷**: PNG (권장)
- **크기**: 128x128 px (필수), 256x256 px (Retina 디스플레이용 권장)
- **배경**: 투명 배경 권장 (브랜드 아이덴티티에 따라 배경색 포함 가능)

### 3. 애플리케이션용 (App Icon)
데스크탑 앱의 독 (Dock) 및 작업 표시줄 아이콘입니다.
- **포맷**: PNG (투명 배경)
- **필수 크기**:
  - 1024x1024 px
  - 512x512 px
  - 256x256 px
  - 32x32 px (Favicon 등 용도)
- **참고**: Windows(.ico) 및 Mac(.icns) 변환을 위해 고해상도 PNG가 가장 중요합니다.

## ✅ 작업 팁 및 고려사항 (Tips)
1. **작은 크기에서의 가독성 (Readability)**
   - 아이콘은 16x16px (VS Code Activity Bar 등) 처럼 매우 작게 표시될 수 있습니다. 복잡한 선이나 디테일은 뭉개질 수 있으니, **단순하고 굵은 형태**가 유리합니다.

2. **배경색 대비 (Contrast)**
   - 많은 개발자가 **다크 모드**를 사용합니다. 어두운 배경에서도 아이콘이 잘 보이는지 확인해주세요. (지나치게 어두운 색상은 피하는 것이 좋습니다)

3. **여백 (Padding)**
   - 꽉 찬 사각형 형태보다는, 가장자리에 약 10~15% 정도의 여백을 두는 것이 시각적으로 안정적입니다.

4. **일관성 (Consistency)**
   - VS Code의 기본 아이콘 스타일(얇은 선, 단색 등)과 어울리면 좋지만, 브랜드의 독자적인 컬러를 사용하여 눈에 띄게 하는 것도 좋은 전략입니다.
